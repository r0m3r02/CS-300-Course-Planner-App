# CS 300 Portfolio Reflection

The projects in this course were centered around finding an efficient way to organize course information and make it easy to access. I built a program that could read data from a file, store it in a data structure, search for individual courses, and display every course in alphabetical order. I also evaluated different data structures to understand why one solution may perform better than another depending on the situation.

At the beginning of the course, I mostly cared about whether my code produced the correct output. As I worked through the projects, I started paying much more attention to what was happening behind the scenes. I learned that choosing the right data structure can have a major impact on how quickly a program runs and how much memory it uses. That changed the way I approached solving programming problems.

One obstacle I faced was working through errors that were not always obvious. Sometimes the program compiled but did not behave correctly, which meant I had to trace the logic instead of just fixing syntax errors. I found that testing one section at a time and checking my assumptions made it much easier to identify problems without creating new ones elsewhere in the program.

These projects also helped me become more thoughtful when designing software. Rather than jumping straight into coding, I now spend more time considering how different parts of a program should interact and how the data will move through the application. Having a plan before writing code has made my programs more organized and easier to build.

Looking back, I think the biggest improvement has been in the quality of my code. I make a greater effort to keep functions focused on a single task, use meaningful names, and organize my code so that future changes are easier to make. Those habits will help me as projects become larger and more complex throughout my career.
